# Анализ веса здоровых доношенных детей 0–6 мес: ресерч и редизайн раздела «Вес»

Создан: 2026-06-01
Статус: research + дизайн-концепция (НЕ план реализации)
Метод: глубокий мульти-агентный ресерч (102 агента, состязательная проверка
голосованием — до 3 голосов на claim; из 25 проверенных 21 подтверждён, 4 убито),
затем 4 сессии критики другой моделью.

Документ решает две задачи:
1. Зафиксировать, **как сейчас работает раздел «Вес»** и в чём его дефект.
2. На авторитетных медицинских источниках определить, **какие показатели веса
   действительно важны** для здорового доношенного ребёнка 0–6 мес на ИВ, и
   как **связать анализ веса с планом кормления** (работает ли план).

> **Дисклеймер.** Это не медицинская рекомендация. Все пороги ниже — из
> публичных рекомендаций (WHO, NICE, ABM, CDC/AAP). Приложение их **отображает
> и помогает интерпретировать**, но не заменяет врача и не назначает докорм.

## Часть 0. Краткая выжимка (для нетерпеливых)

- **Главный валидатор того, что план кормления работает — адекватная прибавка
  веса** + responsive feeding (CDC/AAP: ребёнок «takes what they need… stops when
  full», не заставлять допивать — подтверждено дословно, §2.7). Дословной нормы
  «рост **важнее** фиксированного ml/кг» нет — этот приоритет остаётся синтезом
  (§6.1). Оценивается по двум инструментам ВОЗ: **Weight-for-Age (WHO Child Growth
  Standards 2006)** как «где ребёнок сейчас» и **Weight Velocity (WHO 2009)** как
  «с какой скоростью растёт». Скорость — главный сигнал «работает ли план прямо
  сейчас», позиция — фон.
- Для **индивидуального** ребёнка правильный сигнал — **движение вдоль своего
  коридора** (tracking), а не абсолютный перцентиль. Тревога — **устойчивое
  падение** через перцентильные коридоры (пороги NICE NG75, привязаны к весу при
  рождении), либо текущий вес **ниже P2**.
- Стандарты velocity ВОЗ **feeding-agnostic** — применимы к ребёнку на ИВ
  напрямую (норм-когорта была на ГВ, поэтому флаг «низкая скорость» на ИВ-ребёнке
  **консервативен** — смещён в сторону гипо-, а не гипердиагностики; не каждый
  флаг = истинный, но ложноотрицательных меньше).
- Простое правило «**~20–30 г/сут**» — допустимая перекрёстная проверка
  (из secondary-источников, §4), но **первичен z-score скорости ВОЗ**, а не
  граммы/сутки.
- Первые дни: **>10% потери от веса рождения** (NICE) — порог клинической
  оценки (не автоматического докорма; текущая карточка показывает 7%/10% — §3.2
  предлагает свести к единому >10%). Эпидемиология потери/восстановления
  (ABM #3, Flaherman/NEWT, Macdonald) — про ГВ; ИВ-дети теряют **меньше** и
  восстанавливаются **быстрее** (медиана ~6.5 дн против ~8.3 дн у ГВ — медианы из
  одной когорты Macdonald 2003, **confidence medium / голос 2-1**, см. §2.3).
- **Не взвешивать слишком часто.** NICE: не чаще, чем **раз в неделю** в 1–6 мес
  (ежедневно — только до 1 мес). На коротких интервалах шум весов забивает
  сигнал — не считать г/сут по двум близким замерам.

## Часть 1. Как сейчас работает раздел «Вес»

### 1.1. Архитектура (код)

Раздел живёт под маршрутом `/weight` и состоит из двух страниц + общего движка.

| Слой | Файл | Назначение |
|---|---|---|
| Список | `apps/web/app/features/WeightList/WeightList.tsx` | Список взвешиваний (убыв.), Δ с предыдущим, «день жизни» |
| Ввод | `apps/web/app/features/WeightSheet/WeightSheet.tsx` | Форма (дата + граммы), upsert по дате, edit/delete |
| Аналитика | `apps/web/app/features/WeightAnalytics/WeightAnalytics.tsx` (558 строк) | 3 карточки + адаптация + темп WHO + таблица + 2 графика |
| Напоминание | `apps/web/app/features/WeighInBanner/WeighInBanner.tsx` | «Пора взвесить» при ≥3 днях без замера |
| Табы | `apps/web/app/features/WeightTabs/WeightTabs.tsx` | Переключение «Взвешивания / Аналитика» |
| Движок | `packages/domain/src/who/analytics.ts` (`buildAnalytics`) | z-score, перцентили, velocity — вся математика |
| LMS-математика | `packages/domain/src/who/zscore.ts` | `zFromMeasurement`, `percentileFromZ` (erf-аппр.), обратные |
| Типы | `packages/domain/src/who/analyticsTypes.ts` | `AnalyticsPoint`, `WeightsAnalytics`, `MonthlyVelocity` (цель `@leon/domain/who/types`) — их формы меняют §3.2/§3.3 |
| Lookup | `packages/domain/src/who/lookup.ts` | Поиск LMS WFA по дню, early/monthly velocity |
| Данные | `packages/domain/src/who/data/{wfa,velocity-early,velocity-monthly}.json` | Таблицы ВОЗ (вендорятся) |
| API | `apps/api/src/routes/weights.analytics.ts` | `GET /api/weights/analytics` → `buildAnalytics` |

Исходники ВОЗ лежат и в `docs/` как первоисточник: `wfa-*-zscore/percentiles-expanded-tables.xlsx`,
`wv-*-{1,2,3,4,6}mo-zscore.xlsx`, `wv-{boys,girls}-0-60d.pdf`.

### 1.2. Что сейчас показывается (8 показателей)

1. **Текущий вес** + Δ от прошлого замера (граммы).
2. **Перцентиль по ВОЗ** + z-score (weight-for-age, LMS) + возраст в днях.
3. **Статус-бейдж**: Норма (`−2 ≤ z ≤ 2`) / Низкий вес (`−3 ≤ z < −2`) /
   Дефицит (`z < −3`) / Выше среднего (`2 < z ≤ 3`) / Избыток (`z > 3`).
4. **Карточка «Адаптация» (первые 14 дней)**: % потери от веса рождения;
   текстовый статус (Физиологическая потеря ≤7% / на границе ≤10% / выше нормы);
   WHO early-velocity по группам веса рождения (класс P5–P50+).
5. **Карточка «Темп за завершённый интервал WHO»** (monthly velocity): Δ за
   завершённый месячный интервал, z-score + перцентиль скорости, тренд перцентиля.
6. **Таблица по взвешиваниям**: дата, возраст, вес, «прошло дней», Δ, **г/сут**,
   перцентиль.
7. **График динамики веса** (факт во времени).
8. **График динамики перцентиля** (линии P3/P50/P97).

### 1.3. Что хорошо

- Уже считаются **правильные инструменты ВОЗ** (WFA z-score + два вида velocity),
  с честной LMS-математикой и вендоренными таблицами. Это сильный фундамент —
  переделывать движок не нужно, нужно переосмыслить **подачу и связь с планом**.
- Адаптационный режим (0–14 дней) логически отделён от «зрелого» — это
  согласуется с тем, что ранняя потеря веса физиологична и оценивается иначе.
- Замечание про z-границы −1/−2: они **уже живут в коде** —
  `MonthlyVelocityCard` (`WeightAnalytics.tsx`) уже ветвит `z ≥ −1` / `≥ −2` / `<
  −2`. То есть «Светофор» (§3.1) их **переиспользует/формализует**, а не изобретает
  заново; но оговорка «валидировать до релиза» (§6.3) распространяется и на это
  уже работающее поведение.

### 1.4. В чём дефект (то, что вы назвали «хаотично накиданы показатели»)

**Дефект №1 — раздел описательный, а не оценочный относительно плана.**
Все 8 показателей отвечают на вопрос «**где** ребёнок на кривых ВОЗ», но не на
ваш главный вопрос — «**работает ли текущий план кормления**». Нет ни одного
элемента, который бы связывал прибавку веса с тем, что планировщик обещал.

**Дефект №2 — нет обратной петли «план → вес».**
Планировщик (`packages/domain/src/planning/target.ts` →
`energyCurve.ts`) считает суточную цель в мл из энергии FAO/WHO/UNU 2004
(`kcal/кг/сут × вес × ...`). Вес там — **только вход** (масштабирует цель).
Фактическая прибавка нигде не сравнивается с ожидаемой при соблюдении плана.
Раздел «Вес» и раздел «Кормление» — два не связанных мира.

**Дефект №3 — три несогласованных «языка» скорости.**
В таблице — `г/сут` между произвольными парами замеров; в адаптации — WHO
early-velocity по группам; в «темпе» — WHO monthly z-score. Пользователь видит
три разных способа сказать «как быстро растёт» без единого нарратива и без
указания, какой из них главный.

**Дефект №4 — `gramsPerDay` шумный на коротких интервалах.**
`analytics.ts` делит Δ на «прошло дней» для **любой** пары замеров. На интервале
1–2 дня погрешность бытовых весов (±10–20 г) и нормальные суточные колебания
дают мусорные г/сут. NICE прямо предостерегает от такой over-interpretation.

**Дефект №5 (медицинская неточность) — бейдж «Избыток веса» по WFA.**
Текущий `wfaBadge` (`WeightAnalytics.tsx:97`) при `z > 3` показывает «Избыток
веса», при `2 < z ≤ 3` — «Выше среднего», трактуя **высокий** weight-for-age как
отклонение. По CDC/ВОЗ порог `+2 SD = отклонение` относится к **weight-for-length
/ BMI-for-age**, а не к weight-for-age. Для WFA клинически значим **только
нижний хвост** (−2 SD — underweight, −3 SD — severely underweight). Высокий WFA
сам по себе «избытком» не считается (это оценивается по весу-к-длине). → Эту
ветку нужно убрать/переформулировать (см. §3.2).

**Дефект №6 (медицинская неточность) — пороги ранней потери 7%/10% «as is».**
Адаптационная карточка использует плоские 7% (норма) / 10% (граница). По NICE
NG75 действенный порог — единый **>10%** (повод для клинической оценки). Кроме
того, цифры физиологической потери/восстановления, на которые опирается «норма»,
получены на **детях на ГВ**; для ИВ они смещены (ИВ теряет меньше, возвращает
вес быстрее). Это стоит явно учесть в текстах (см. §2.3, §3.2).

## Часть 2. Ресерч: что говорят авторитетные источники

> Источники строго ограничены: WHO, AAP/HealthyChildren, NICE (UK), ABM,
> CDC, FAO/WHO/UNU, и рецензируемые работы признанных авторов (de Onis,
> Flaherman, Macdonald). Каждый факт прошёл состязательную проверку
> (3 независимых голоса; убитые claim'ы вынесены в §5). Годы публикации указаны.

### 2.1. WHO Child Growth Standards 2006 — Weight-for-Age (где ребёнок сейчас)

- **Это «стандарт», а не «референс».** MGRS-исследование (~8 440 детей, 6 стран,
  1997–2003) использовало **prescriptive** подход: кривые описывают, как ребёнок
  должен расти в оптимальных условиях, а не как «в среднем растут» в популяции.
  Опубликовано 11.11.2006. Есть отдельная подвыборка графиков **«Birth to 6
  months»** — ровно охват приложения. *(WHO Child Growth Standards 2006;
  who.int/tools/child-growth-standards)*
- **Для индивидуума статус выражается прежде всего z-score.** WHO определяет
  отклонение через **±2 SD** (математически это 2.3-й и 97.7-й перцентили, но на
  графиках ВОЗ подписаны как **2-й и 98-й**). Для **weight-for-age** клинически
  важен **нижний** хвост: **−2 SD = underweight**, **−3 SD = severely
  underweight**. Высокий WFA сам по себе НЕ флагуется как отклонение (избыток
  оценивают по weight-for-length / BMI). *(CDC «Using the WHO Growth Charts»;
  MMWR RR-59(9) 2010, Grummer-Strawn et al.)*
- **Перцентили vs z-score.** WHO даёт и то, и другое, но z-score предпочтительнее
  для индивидуального мониторинга и для краёв распределения (за P3/P97 перцентиль
  «сжимается», а z продолжает различать −2.5 от −3.5). Приложение уже считает z —
  это правильный первичный язык.
- **Выбор графиков 0–24 мес: WHO, не CDC.** CDC (и AAP/NCHS) с 2010 г.
  рекомендуют для детей **0–2 лет** использовать кривые **WHO**, переключаясь на
  CDC-референс в 24 мес. Для Leon (0–6 мес) WHO — правильный авторитет (он уже
  вендорится). *(CDC; MMWR RR-59(9) 2010)*

### 2.2. WHO Weight Velocity Standards 2009 — скорость набора (работает ли план сейчас)

- **Структура таблиц** (de Onis et al., MGRS, ISBN 9789241547635,
  handle 10665/44026): **1-месячные** инкременты веса от рождения до 12 мес
  (z-score + перцентили, LMS); инкременты **2/3/4/6-месячные** до 24 мес
  («2–6 месяцев» — это дискретный набор {2,3,4,6}, **5-месячной таблицы нет**);
  **ранние** инкременты **по группам веса рождения** в интервалах
  **0–7, 7–14, 14–28, 28–42, 42–60 дней** (1- и 2-недельные). Группы веса
  рождения: 2000–2500, 2500–3000, 3000–3500, 3500–4000, 4000+ г, мальчики/девочки
  раздельно, в граммах. *(WHO 2009; WHO/PAHO Annex 3, NBK148960)*
- **Feeding-agnostic.** WHO прямо: стандарты «can be used to assess children
  everywhere, regardless of ethnicity, socioeconomic status and **type of
  feeding**». → применимы к ИВ-ребёнку напрямую. Норм-когорта MGRS была на ГВ, и
  ИВ-дети после ~3 мес склонны **превышать** velocity — поэтому флаг **низкой**
  скорости у ИВ-ребёнка **консервативен** (настоящий повод смотреть, не артефакт).
- **Как читать.** Инкремент за интервал → z-score скорости. **Низкий** z
  (например, ниже −1…−2 по нижним перцентилям) = недостаточный темп → повод
  пересмотреть план/обследовать. Это и есть «работает ли план **в динамике**».
- **Важная оговорка по early-таблицам.** Ранние инкременты по группам веса
  рождения — это **эмпирические перцентили** (медиана/25/10/5), **не** модельный
  GAMLSS z-score. Малые группы (в полных таблицах ВОЗ): 2000–2500 г — n=7
  мальчиков; 4000+ — n=46. → трактовать ранние числа как описательные медианы, не
  как прескриптивные z-cutoff'ы. *(WHO 2009)* Примечание: группа 2000–2500 в
  данных приложения **не вендорится** (см. §4) — она вне scope «здоровых
  доношенных».

**Конкретные ранние медианы (граммы, ВОЗ):** день 0–7 — самый низкий недельный
инкремент (~150 г мальчики в большинстве групп, лишь **50 г** для 4000+; девочки
~100–150 г по вендоренным группам — в полных таблицах ВОЗ нижняя группа даёт 0 г,
но её приложение не использует) — отражает физиологический послеродовой надир.
День 7–14 → ~250–275 г (мальчики) / ~200 г (девочки). День 14–28 → ~600–725 г
(мальчики). В пересчёте недели 2–4 ≈ **36–52 г/сут** (250 г/7 ≈ 36 … 725 г/14 ≈
52; в сыром артефакте ресерча округлено как 36–46), что согласуется с (и выше)
правилом «20–30 г/сут». *(WHO Growth Velocity Standards 2009; NBK148960)*

### 2.2a. Поведение ИВ-ребёнка на кривых ВОЗ (важно для трактовки скорости)

Норм-когорта ВОЗ была на ГВ, поэтому ИВ-ребёнок систематически смещён относительно
кривых — и это **подтверждённый CDC факт**, а не наша догадка (закрывает прежний
открытый вопрос про «слишком быстрый набор»):

- **После ~3 мес ИВ-дети набирают быстрее и пересекают коридоры ВВЕРХ.** CDC MMWR
  RR-59(9) 2010 дословно: «*formula-fed infants tend to gain weight more rapidly
  after approximately age 3 months and therefore **cross upward in percentiles**,
  perhaps becoming classified as overweight*». То же на training-странице CDC:
  «…may be more likely to **cross upward in percentiles**… perhaps becoming
  classified as **high weight for length**».
- **В первые ~3 мес — наоборот, могут выглядеть медленнее** относительно кривых
  ВОЗ (которые отражают более быстрый ранний набор у ГВ): «*Healthy breastfed
  infants typically gain weight faster than formula-fed infants in the first few
  months… then gain weight more slowly*».

**Продуктовое следствие (важно для «Светофора»):** не гнать ИВ-ребёнка к медиане
ВОЗ любой ценой. Слегка более низкий темп **до 3 мес** и более быстрый набор
**после 3 мес** — это **ожидаемо**, а не «план сломался / план перевыполнен».
Восходящее пересечение коридоров у ИВ после 3 мес само по себе не должно
поднимать тревогу без weight-for-length и клинического контекста.
*(CDC MMWR RR-59(9) 2010; CDC Using the WHO Growth Charts)*

### 2.3. Ранняя потеря веса и восстановление (первые дни/недели)

- **Физиология.** Ранняя потеря веса нормальна (диурез внеклеточной жидкости +
  отхождение мекония). *(StatPearls NBK536449)*
- **Действенный порог — >10% от веса рождения (NICE NG75, Rec 1.1.2):** при
  потере >10% «provide clinical support... perform a clinical assessment, looking
  for evidence of dehydration... take a detailed history to assess feeding». Это
  **повод оценить**, а НЕ автоматический докорм.
- **ABM Clinical Protocol #3 (ред. 2017):** «≥8–10% (день 5 / 120 ч или позже),
  или потеря выше **75-го перцентиля** для возраста... показание к careful
  assessment». И прямо: «Ten percent weight loss is **not** an automatic marker
  for the need for supplementation, but **is an indicator for infant
  evaluation**». 75-й перцентиль берётся из номограммы **NEWT**
  (newbornweight.org, Flaherman et al., Pediatrics 2015, n=108 907 ГВ).
- **Надир потери не фиксирован и зависит от способа родов:** медиана надира у ГВ
  ~7.1% в 48 ч (вагинальные роды), ~8.6% в 72 ч (кесарево); ~5% вагинальных vs
  ~10% кесаревых превышают 10% к 48 ч (>25% кесаревых к 72 ч). → **не хардкодить
  единый «день надира».** *(Flaherman 2015; claim о «надир день 2–3 независимо от
  способа родов» был ОПРОВЕРГНУТ — см. §5)*
- **Восстановление веса рождения:** у ГВ медиана ~**8.3 дня** (95% CI 7.7–8.9),
  97.5% — к **21 дню**; NICE: «most infants have returned to their birth weight
  by 3 weeks of age», и отсутствие восстановления к 3 неделям → рассмотреть
  направление к врачу. **ИВ-дети восстанавливаются быстрее: медиана ~6.5 дня,
  97.5-й перцентиль ~16.7 дня** (та же когорта Macdonald 2003, Arch Dis Child
  Fetal Neonatal Ed 2003;88:F472). → **не применять ГВ-пороги к ИВ без поправки.**

> **Граница применимости (важно для Leon).** Самая богатая эпидемиология ранней
> потери/восстановления (ABM #3, Flaherman/NEWT, Macdonald) — **про ГВ** и из
> Baby-Friendly/исключительно-грудных когорт. Leon — про **ИВ**. Поэтому:
> (i) **стандарты WHO velocity и WFA feeding-agnostic** — применяй напрямую;
> (ii) **порог NICE >10%** и **пороги faltering growth NICE** применяются ко
> **всем** детям независимо от типа вскармливания; (iii) конкретные «нормальные»
> числа потери/дней восстановления для ИВ — **меньше/быстрее**, это надо отражать
> в текстах.

### 2.4. «Адекватный рост» и «faltering growth» (0–6 мес)

- **Faltering growth** — авторитетный преемник термина «failure to thrive».
  NICE NG75 / QS197 операционализирует через **устойчивое падение по
  перцентильным коридорам**, порог привязан к перцентилю **веса при рождении**
  (NG75, Rec 1.2.1):
  - падение через **≥1** коридор, если вес рождения был **<9-го** перцентиля;
  - падение через **≥2** коридора, если вес рождения **9-й…91-й** перцентиль;
  - падение через **≥3** коридора, если вес рождения **>91-го** перцентиля;
  - **текущий вес <2-го перцентиля** для возраста — при любом весе рождения.
- NICE формулирует это как **«thresholds for concern»** (поводы для оценки), а
  не как жёсткий диагноз. «Перцентильный коридор» (centile space) на UK-графиках —
  это промежуток между соседними напечатанными перцентильными линиями (напр.,
  9-й→25-й). *(NICE NG75 2017; QS197 2020; StatPearls NBK536449)*
- **Источник порогов — преимущественно NICE (UK).** Эквивалентно явных численных
  порогов «коридор/каденс» от AAP или ESPGHAN в проверенных claim'ах **не
  нашлось**. → количественная логика faltering growth и каденса опирается на NICE.

### 2.5. Правила «граммы/сутки» по возрасту (перекрёстная проверка)

- Эмпирические «правила большого пальца» (~**20–30 г/сут** в первые месяцы)
  существуют в педиатрических источниках (AAP/HealthyChildren, MSD Manual —
  **это secondary-источники**, см. §4) и **согласуются** с медианами WHO для
  недель 2–4 (~**36–52 г/сут**: день 7–14 ≈ 250 г/7 ≈ 36; день 14–28 у мальчиков
  до 725 г/14 ≈ 52). Сами числа «20–30 г/сут» — из secondary-источников; это
  **эмпирические медианы с малыми группами**, описательно, не как z-cutoff.
- **Но первичен z-score скорости ВОЗ**, а граммы/сутки — вторичная, понятная
  пользователю перекрёстная проверка. Граммы/сутки не учитывают пол, возраст и
  форму кривой; z-score ВОЗ учитывает всё это.
- Терминологическая оговорка: здесь **два разных смысла слова «вторичный»**.
  (1) По **аналитике**: WHO velocity — первичный показатель, г/сут — вторичная
  сверка. (2) По **качеству источника**: сами числа г/сут взяты из
  **secondary-источников** (§4), тогда как медианы WHO — primary. Оба верны и не
  противоречат друг другу. *(приоритет «velocity → primary, г/сут → secondary»
  запрошен и подтверждён ресерчем как аналитическое требование.)*

### 2.6. Каденс взвешивания и качество измерения

- **NICE NG75 (Rec 1.2.27)** ограничивает частоту: «usually no more often than:
  **daily if less than 1 month**; **weekly between 1–6 months**; fortnightly
  6–12 мес; monthly с 1 года».
- **NICE (Rec 1.2.28)** прямо предупреждает: взвешивание чаще рекомендованного
  «may increase parental anxiety over minor fluctuations». → приложение должно
  **не поощрять** слишком частые замеры и **не считать г/сут на слишком коротких
  интервалах** (там доминирует шум весов).
- Это «soft maximum» («usually no more often than»), а не жёсткий запрет —
  учитывает уровень тревоги/показания (claim с формулировкой «жёсткий запрет
  чаще недели» был ОПРОВЕРГНУТ, см. §5). Подаём как **рекомендацию/оговорку**.

### 2.7. Связь «поступление → рост» (центральный вопрос Q7)

**Что удалось установить (синтез из единогласно подтверждённых кусков):**
авторитетная рамка поддерживает использование **адекватной прибавки веса**
(по WFA z-score + velocity z-score ВОЗ) как **первичного подтверждения, что
объёма смеси достаточно**, а не жёсткого «фиксированного ml/кг». Лестница
эскалации:
- **(a)** нормальная скорость / движение вдоль коридора → **план работает**;
- **(b)** ранняя потеря >10%, ИЛИ низкая/падающая скорость, ИЛИ устойчивое
  падение через коридоры (пороги NICE по весу рождения) → **пересмотр плана
  кормления / оценка**;
- **(c)** текущий вес <P2, невосстановление веса рождения к 3 неделям, либо
  выраженный устойчивый faltering → **направление к врачу**.

**Опора на responsive feeding (проверено дословно, CDC + AAP).** Идея «объём —
стартовая модель, а подтверждает её рост + поведение ребёнка, а не жёсткий ml/кг»
теперь стоит на прямых цитатах, а не только на синтезе:
- CDC: «*Babies generally **take what they need** at each feeding and **stop
  eating when full**.*» *(How Much and How Often to Feed Infant Formula)*
- CDC: «*Watch your baby for cues that they are full, and then **stop, even if the
  bottle is not empty**.*» / «*Do **not force** your baby to finish the bottle…
  This can lead to your baby **eating more than they need**.*» *(About Feeding
  From a Bottle)*
- AAP/HealthyChildren: «*…they probably will **regulate their intake** from day to
  day to meet their own specific needs, so **let them tell you when they've had
  enough**.*» *(Amount and Schedule of Formula Feedings)*

**Что проверять при плохой прибавке — feeding audit, а не авто-рост объёма
(NICE NG75, Rec 1.2.10, дословно).** «*…consider whether any of the following are
contributing to faltering growth in milk-fed infants: …**ineffective bottle
feeding**; feeding patterns or routines…; the feeding environment; feeding
aversion; parent/carer–infant interactions; how parents or carers **respond to the
infant's feeding cues**; physical disorders that affect feeding.*» → «жёлтая»
ветка «Светофора» (§3.1) должна вести к проверке **приготовления смеси, техники
кормления из бутылочки, ритма, реакции на cues**, и лишь потом — к пересмотру цели.

> **Остаточная оговорка (gap сузился).** Прямая авторитетная **responsive-feeding**
> рамка теперь зафиксирована (CDC + AAP + NICE 1.2.10 выше) — это уже **не
> синтез**. Чего по-прежнему нет дословно — это ровно одного предложения вида
> «прибавка веса валидирует объём смеси **лучше**, чем соблюдение фиксированного
> ml/кг». Эту конкретную формулировку (приоритет роста **над** объёмом) ни один
> источник так не ставит — она остаётся **обоснованным синтезом** (confidence:
> medium→ближе к supported). Подаём как «рост + responsive feeding подтверждают
> достаточность», не как «можно игнорировать объём». ESPGHAN-формулировку про
> responsive feeding всё ещё полезно дозабрать (§6.1).

## Часть 3. Предлагаемый редизайн раздела «Вес»

Принцип: **меньше описательных чисел, больше ответов на вопрос «всё ли ок с
планом»**. Один главный нарратив (растёт ли ребёнок по своему коридору с
адекватной скоростью), под ним — детали по запросу.

### 3.1. Главный экран — индикатор роста (новое, key change)

Один **верхний блок-вердикт**, агрегирующий сигналы. Цвет (🟢/🟡/🔴) — визуал, но
**подписи намеренно наблюдательные, не директивные**: вес — это outcome, а не
доказательство «идеального плана». Поэтому НЕ «план работает / объём адекватен», а
«рост идёт без сигналов тревоги».

> 🩺 **Программная формулировка (главный принцип раздела).** Вес не доказывает
> идеальность плана, но является **главным внешним валидатором** того, что ребёнок
> получает достаточно питания. Объём смеси — **расчётная цель**; рост, поведение
> ребёнка (responsive feeding, §2.7) и отсутствие тревожных признаков — **проверка
> реальностью**. Нормальный вес подтверждает «вероятно достаточно», но не
> исключает перекорм, задержку жидкости или ошибку измерения — поэтому формулировки
> мягкие.

> ⚠️ **z-границы −1 / −2 — инженерная эвристика, НЕ порог ВОЗ/NICE** (§6.3). ВОЗ
> даёт перцентили скорости, но не называет z-cutoff «тревоги». До клинической
> валидации velocity-z используется как **supporting signal**, а не как
> самостоятельный диагноз: один лишь `velocity z < −2` **не должен** в одиночку
> давать 🔴 — он повышает статус до 🟡 «перепроверь», а 🔴 ставится только в связке
> с другим сигналом (см. ниже). **Velocity-z** (2009) и **сползание по коридорам
> WFA** (tracking, 2006) — два РАЗНЫХ инструмента; при расхождении приоритет
> открыт (§6.3).

- 🟢 **Рост идёт без сигналов тревоги.** Ребёнок движется вдоль своего коридора
  WFA; скорость в норме (velocity z ≥ −1 — *эвристика, §6.3*). Подпись: «вес
  растёт ожидаемо — **текущий режим, вероятно, достаточен**». (Зелёный даже при
  дефиците кормления — рост это внешний валидатор; см. §3.3.)
- 🟡 **Есть повод перепроверить кормление/измерения.** Любой из: velocity z в
  [−2, −1) (*эвристика*); **velocity z < −2 сам по себе** (без других сигналов —
  тогда это «перепроверь», не «к врачу»); устойчивое падение через коридоры **по
  порогам NICE, привязанным к весу рождения** (§2.4: для веса рождения 9–91 перц.
  это **≥2** коридора, не 1); в адаптации — потеря близко к 10%; **быстрый набор**
  (для ИВ восходящее пересечение после 3 мес **ожидаемо**, §2.2a — подпись
  «быстрый набор; для оценки избытка нужна **длина** (weight-for-length, §6.7)», НЕ
  «перекорм»). Подпись: «прибавка вне ожидаемого — проверь измерение и кормление».
  **Действие при низкой прибавке — feeding audit, а не сразу +объём** (NICE 1.2.10,
  §2.7): приготовление смеси, техника бутылочки, ритм, реакция на cues; затем
  перевзвесь через неделю.
- 🔴 **Нужна консультация врача.** Любой из (это **клинические** пороги, не
  эвристика): устойчивое падение через коридоры **сверх порога NICE** (§2.4);
  текущий вес < P2; ранняя потеря > 10%; невосстановление веса рождения к 3
  неделям. (Низкая velocity сюда добавляется **только в связке** с одним из этих —
  сама по себе она даёт 🟡, см. оговорку выше.) Подпись: «покажись педиатру; это
  сигнал, а не диагноз».

**Граничные состояния (обязательно обработать):**
- **< 2 взвешиваний** или первое взвешивание: скорости/тренда нет
  (`analytics.ts` возвращает `null` для первой точки) → светофор не строим,
  показываем только текущий вес/перцентиль.
- **Адаптация (0–13 дней):** плановой цели нет (`computeTarget` → `null` в
  neonatal-режиме), значит **дефицита кормления тоже нет** — ветки светофора,
  завязанные на дефицит, в этом окне не работают; опираемся на потерю % и early
  velocity.
- **Дефицит = null** (план ещё не настроен / интервал старше плана): откатываемся
  на вердикт **только по скорости/тренду**, без оси дефицита.
- **Запаздывание скорости (важно для смысла «работает ли план *сейчас*»).**
  Месячная velocity (`monthlyVelocity`) — это **завершённый** интервал WHO, он
  отстаёт от «сегодня» **до ~30 дней**. Дефицит же может быть свежим (последние
  дни). Значит ребёнок с хорошей velocity за прошлый завершённый месяц, но с
  планом, развалившимся на этой неделе, иначе попал бы в 🟢. Поэтому: при
  **большом свежем дефиците + зелёной velocity** не давать чистый зелёный —
  мягко предупреждать «план недавно не выдерживается, прибавку это пока не
  отразило». (Полностью снимется, если velocity считать по интервалу,
  выровненному со взвешиваниями — это net-new, см. §3.3/Прил.A.)

Этот блок — **единственное место, где «Вес» прямо отвечает на ваш вопрос**.
Остальное (карточки, таблица, графики) становится «деталями под катом».
Замечание про каденс: 🟡 «взвесь через неделю» согласуется с каденсом NICE
(§2.6) — не подталкиваем взвешиваться чаще и тем самым не раздуваем тревогу.

### 3.3. Что добавить (связь с планом)

> **Напоминание (Q7, medium confidence).** Весь тезис «прибавка веса = валидатор
> плана» — это **синтез ресерча, не дословная норма** (§2.7, §6.1). Прямой цитаты
> AAP/ESPGHAN про «рост вместо фиксированного ml/кг» пока нет. Всё в §3.3 строим
> на этой обоснованной, но непроверенной до конца связке — до релиза дозабрать
> цитату.

- **Сопоставление «ожидаемая vs фактическая прибавка».** Используем **уже
  посчитанный** `monthlyVelocity` (завершённый интервал WHO, с `z`/перцентилем),
  а НЕ произвольный интервал между двумя замерами: таблицы velocity ВОЗ дискретны
  ({1,2,3,4,6 мес}) и для интервалов < 14 дней не определены. Показать: «WHO-темп
  за завершённый месячный интервал: z=…; у Леона столько-то г». *(Сопоставление
  произвольного интервала между замерами с velocity ВОЗ не поддерживается —
  `lookupVelocityLMS` существует, но `buildAnalytics` его не вызывает; см. §6.)*
- **Связка с дефицитом кормления (как гипотеза, не причинность).** Если копится
  дефицит (`target − factOfDay`, `metrics.ts`) И скорость падает — это **усиливает
  гипотезу**, что поступление недостаточно (а не доказывает «отсюда низкая
  прибавка» — каузальность данные не подтверждают). Если дефицита нет, а прибавка
  низкая — гипотеза смещается к «план выдержан, но роста нет → перепроверь
  измерение, затем к врачу». Эти случаи **наводят на разные действия**, и сейчас
  приложение их не различает.
  > **Цена интеграции (честно).** Дефицит сейчас считается **подневно** в
  > `dayMetrics` и отдаётся **только через `/api/history`** (там в ответ идёт
  > `...m` — полные метрики). Эндпоинт `/api/feedings/analytics` возвращает
  > `target`/`fact`/`mode`, но `deficit` **отбрасывает** (проекция
  > `fact: m.factOfDay`); `weights.analytics` о кормлениях **ничего не знает**
  > вовсе. То есть в виде, выровненном по интервалу между взвешиваниями, дефицита
  > нет нигде. «Средний
  > дефицит за интервал между взвешиваниями» — **новый расчёт, которого в коде
  > нет**: нужно объединить диапазон истории кормления с интервалом между замерами
  > (расширение `buildAnalytics` или новый эндпоинт). То есть «движок уже даёт всё»
  > верно для **весовой** стороны (velocity-z, WFA-z) и **кормительной** стороны
  > по отдельности, но **join** между ними — net-new и нетривиальный.
- **Напоминание о взвешивании** привести к каденсу NICE: для 0–1 мес допускать
  чаще, для 1–6 мес — целить в **~раз в неделю** (сейчас баннер на ≥3 дня — это
  ок как нижняя граница, но текст стоит согласовать с «не чаще недели полезно»).

### 3.4. Чего НЕ делать (anti-requirements из ресерча)

- Не флагать высокий weight-for-age как «избыток» — избыток это weight-for-length
  > P98 (§2.1, §2.2a, §3.2).
- **Не пугать восходящим пересечением коридоров у ИВ после 3 мес — это ожидаемо**
  (CDC, §2.2a); и не хвалить быстрый набор как «успех плана» без weight-for-length.
- Не хардкодить единый «день надира» потери (§2.3).
- Не применять ГВ-числа потери/восстановления к ИВ без оговорки (§2.3).
- Не считать темп по слишком частым замерам и не подталкивать к ним (§2.6).
- Не превращать «потеря 10%» в авто-докорм — это повод **оценить** (§2.3).
- При низкой прибавке — сначала **feeding audit** (NICE 1.2.10), а не авто-+объём
  (§2.7).
- Не навязывать жёсткий ml/кг как самоцель — валидатор это **рост + responsive
  feeding** (CDC/AAP подтверждено, §2.7); но дословной нормы «рост важнее объёма»
  нет — не подавать как абсолют (§6.1).

## Часть 5. Опровергнутые утверждения (что НЕ закладывать)

Состязательная проверка **убила** 4 правдоподобных, но неподтверждённых тезиса —
фиксирую, чтобы они не просочились в реализацию:

1. ❌ «Надир потери — день 2–3, **независимо** от способа родов/типа вскармливания»
   (1-2). На деле надир зависит от способа родов (вагинальные ~48 ч, кесарево
   ~72 ч+). *Не хардкодить единый день надира.*
2. ❌ «**Большинство** детей возвращают вес рождения к 21 дню» как общее правило
   (0-3). 21 день — это **97.5-й перцентиль** (верхняя граница), а не «медиана/
   большинство». Медиана у ГВ ~8.3 дн, у ИВ ~6.5 дн.
3. ❌ «AAP RCT (Hospital Pediatrics 2022) операционализировал мониторинг через
   75-й перцентиль NEWT» (0-3) — источник не подтвердил эту атрибуцию. (75-й
   перцентиль как критерий есть у **ABM #3**, не у этого RCT.)
4. ❌ «NICE **запрещает** взвешивать чаще раза в неделю в 1–6 мес» как жёсткое
   правило (0-3). NICE формулирует «usually no more often than» — это **мягкий
   максимум** с учётом уровня концерна, не абсолютный запрет.

## Часть 6. Открытые вопросы (для следующего ресерча/реализации)

### 6.1. Q7 — частично закрыт (осталась ESPGHAN + точная формулировка приоритета)
**Закрыто:** responsive-feeding рамка теперь подтверждена дословно — CDC (×2) +
AAP/HealthyChildren + NICE NG75 Rec 1.2.10 (см. §2.7). **Осталось:** (1) дозабрать
формулировку **ESPGHAN** про responsive feeding (для полноты состава источников);
(2) ни один источник дословно не ставит «прибавка валидирует объём **лучше**, чем
фиксированный ml/кг» — этот **приоритет роста над объёмом** остаётся обоснованным
синтезом, не дословной нормой.

### 6.2. Поправка velocity для ИВ (порог «слишком быстро») — направление закрыто
**Закрыто (CDC, §2.2a):** качественно подтверждено — ИВ-дети после ~3 мес набирают
быстрее и **пересекают коридоры вверх** (это ожидаемо, не тревога), а до 3 мес
могут выглядеть медленнее кривых ВОЗ. **Осталось:** численный порог — с какого
именно темпа/z восходящее пересечение у ИВ становится поводом для weight-for-length
/ оценки перекорма. CDC даёт направление, но не число. → ветка «высокая скорость →
🟡» в §3.1/Прил.A пока без **численного** сорсированного порога.

### 6.3. Коридоры NICE ↔ z-score ВОЗ
NICE выражает faltering growth в «перцентильных коридорах» (UK-конвенция), а Leon
вендорит WHO LMS/z-score. Нужен авторитетный z-эквивалент «падение на 2 коридора»
на кривых WHO (есть ли у WHO/de Onis рекомендованный порог изменения z для WFA,
который мапится на правило NICE). От этого зависят z-границы −1/−2 в §3.1.

### 6.4. Шум весов / минимальный значимый Δ
За пределами каденса NICE и оговорки про тревогу не нашлось авторитетного
источника, **количественно** задающего нормальные суточные колебания и
минимальную точность бытовых весов. → правило «минимальный интервал для валидного
г/сут» и «минимальный значимый Δ» пока без прямой цитаты (поставлено эвристически:
≥7 дней).

### 6.5. Окно день-14 → первый завершённый месячный интервал WHO
После выхода из адаптации (день 14), но до первого завершённого 1-месячного
интервала velocity (`lookupCompletedMonthlyVelocityLMS` возвращает `null`, пока
`ageDays` < `endDays` первого интервала), у «Светофора» **нет оси скорости**.
Нужно явно задать, что он показывает в этом окне (вероятно: только WFA-перцентиль
+ ранняя velocity, без месячного z). Сейчас этот state в §3.1 не назван.

### 6.6. Доношенный, но <2500 г (SGA/LBW term)
Приложение целит в «здорового доношенного», но доношенный ребёнок может быть
<2500 г. `birthWeightGroup()` маппит `<2500 → "2000-2500"`, а этой группы **нет в
вендоренных** `velocity-early.json` → `lookupCompletedEarlyVelocity` молча вернёт
`null` (early-velocity пропадёт). Это **осознанное продуктовое решение** (вне
scope) или пробел? Если осознанное — зафиксировать как известное ограничение; если
нет — довендорить группу 2000–2500.

### 6.7. Weight-for-length пока не считается (нужен для сигнала перекорма)
Авторитетный показатель **избытка массы** у детей <2 лет — это **weight-for-length
≥ P97.7** (CDC, §2.2a/§3.2), а НЕ высокий weight-for-age. Сейчас Leon
weight-for-length **не вычисляет** (вендорятся только WFA + velocity). Если нужен
честный сигнал «перекорм/слишком быстрый набор» (особенно для ИВ после 3 мес),
нужно **довендорить таблицы weight-for-length ВОЗ** (LMS, 0–2 года, по полу) и
добавить расчёт по аналогии с WFA. Без него ветка «высокая скорость» в «Светофоре»
не имеет корректного подтверждающего показателя.

## Приложение A. Связь с планировщиком (как технически переиспользовать)

Движок даёт «весовую» и «кормительную» стороны **по отдельности**; сам **join**
между ними — net-new (см. цену интеграции в §3.3):

- `packages/domain/src/planning/energyCurve.ts` — `targetKcalPerKg(ageMonths)`
  (FAO/WHO/UNU 2004): целевая энергия по возрасту.
- `packages/domain/src/planning/target.ts` — `computeTarget(...)` → суточная цель
  в мл (energy mode; в neonatal-режиме 0–13 дн → **`null`**!) и
  `computeFeedingGuidance(...)` → диапазоны, флаги.
- `packages/domain/src/planning/metrics.ts` — `dayMetrics(...)` → `factOfDay`,
  `deficit = target − factOfDay` (**подневно**; при `target == null` → `null`).
  `deficit` отдаётся **только через `/api/history`** (`...m`); `/api/feedings/
  analytics` его **отбрасывает** (возвращает `fact`, не `deficit`);
  `weights.analytics` о кормлениях не знает.
- `packages/domain/src/who/analytics.ts` — `buildAnalytics(...)` → `monthlyVelocity`
  (z/перцентиль) за **завершённый** интервал WHO. ⚠️ Для произвольного интервала
  между замерами velocity-z **не считается**: `lookupVelocityLMS` существует, но
  `buildAnalytics` его **не вызывает**, и для интервалов < 14 дней он → `null`.

**Идея связки (без кода, концепт):** сопоставить (1) **фактическую** прибавку,
(2) **ожидаемую** по WHO velocity (через завершённый интервал, не произвольный),
(3) **исполнение плана** = средний дефицит за интервал (расчёт net-new). Логика
для «Светофора» (§3.1). **z-границы −1/−2 — эвристика (§6.3), не норма ВОЗ.**

Подписи — наблюдательные, выводы — как **гипотезы**, не причинность (см. §3.1):

| Прибавка | Дефицит кормления | Гипотеза / подпись |
|---|---|---|
| норм (z≥−1 *эвр.*) | любой (в т.ч. большой) | 🟢 рост без сигналов тревоги — *режим вероятно достаточен; рост это валидатор, поэтому зелёный даже при дефиците* |
| низкая (z<−1 *эвр.*) | большой | 🟡 *дефицит + слабая динамика усиливают гипотезу «поступления мало»* → проверь измерение и кормление, перевзвесь через неделю |
| низкая (z<−1 *эвр.*) | ~нулевой | 🟡 *план выдержан, а роста нет — сначала перепроверь измерение/cues; если подтверждается — к врачу* (низкая velocity сама по себе ≠ 🔴) |
| низкая (z<−1 *эвр.*) | **null** (плана не было) | 🟡 откат на вердикт только по скорости — оси дефицита нет |
| **высокая** (быстрый набор) | любой | 🟡 *быстрый набор; для оценки избытка нужна **длина** (weight-for-length, §6.7)* — НЕ «перекорм»; для ИВ после 3 мес это ожидаемо (§2.2a) |
| любая | адаптация 0–13 дн | дефицита нет (target=null) → судим по потере % + early velocity, не по этой таблице |

🔴 в этой таблице **не появляется от одной velocity** — он ставится клиническими
порогами (§3.1: <P2, потеря >10%, падение сверх NICE, нет регейна к 3 нед), а
низкая скорость лишь усиливает их.

Это и есть недостающая обратная петля, ради которой затевался редизайн — но её
**ядро (interval-aligned join + neonatal-aware) предстоит построить**, движок
сам по себе её ещё не даёт.
